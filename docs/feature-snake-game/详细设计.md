### 1.2 Snake (`snake.py`)

**定位**: 纯逻辑组件，无 Pygame 依赖。管理蛇身坐标和移动方向。

```python
class Snake:
```

| 成员 | 签名 | 说明 |
|------|------|------|
| **属性** | | |
| `body` | `List[Tuple[int, int]]` | 蛇身坐标列表，索引 0 为蛇头，末尾为蛇尾 |
| `direction` | `Tuple[int, int]` | 当前移动方向向量：(1,0)右, (-1,0)左, (0,-1)上, (0,1)下 |
| `_grid_cols` | `int` | 网格列数（构造时注入，用于 reset 时计算初始位置） |
| `_grid_rows` | `int` | 网格行数 |
| `_initial_length` | `int` | 初始长度（用于 reset） |
| **计算属性** | | |
| `head` | `@property -> Tuple[int, int]` | 返回 `body[0]`，即蛇头坐标 |
| `length` | `@property -> int` | 返回 `len(body)` |
| **方法** | | |
| `__init__` | `(self, grid_cols: int, grid_rows: int, initial_length: int = 3) -> None` | 初始化蛇身：在网格水平中心生成连续蛇身，蛇头在最右，默认方向向右。蛇身 y 坐标为 `grid_rows // 2`，x 坐标从 `grid_cols // 2 - initial_length + 1` 到 `grid_cols // 2` |
| `move()` | `(self) -> None` | 计算 `new_head = (head[0]+direction[0], head[1]+direction[1])`，执行 `body.insert(0, new_head)` + `body.pop()`。注意：调用者需在调用 move 前通过 grow() 标记本帧是否增长（见 grow 的实现策略） |
| `move_and_grow()` | `(self, grow_flag: bool) -> None` | 执行 `body.insert(0, new_head)`；若 `grow_flag=False` 则 `body.pop()`，否则跳过 pop 实现增长。此方法将移动+增长二合一，避免因 move/grow 分离导致逻辑顺序错误 |
| `change_direction()` | `(self, dx: int, dy: int) -> bool` | 若 `(dx, dy) != (-self.direction[0], -self.direction[1])`（即非反向），更新 `self.direction = (dx, dy)` 并返回 `True`；否则忽略，返回 `False` |
| `check_boundary_collision()` | `(self, cols: int, rows: int) -> bool` | 若 `head[0] < 0 or head[0] >= cols or head[1] < 0 or head[1] >= rows` 则返回 `True`；否则 `False` |
| `check_self_collision()` | `(self) -> bool` | 若 `head in body[1:]` 则返回 `True`；否则 `False` |
| `reset()` | `(self) -> None` | 恢复 `body` 为初始位置，`direction` 重置为 `(1, 0)` |

**`__init__` 初始位置计算公式**：
```
head_x = grid_cols // 2
head_y = grid_rows // 2
body = [(head_x - i, head_y) for i in range(initial_length - 1, -1, -1)]
# 结果：body[0] = (head_x, head_y) 为蛇头，body[2] = (head_x-2, head_y) 为蛇尾
```

### 1.3 Food (`food.py`)

**定位**: 纯逻辑组件，仅依赖 Python `random` 标准库。

```python
class Food:
```

| 成员 | 签名 | 说明 |
|------|------|------|
| **属性** | | |
| `position` | `Tuple[int, int]` | 食物当前坐标。初始值为 `(-1, -1)` 占位，`respawn` 调用后赋值 |
| `_grid_cols` | `int` | 网格列数 |
| `_grid_rows` | `int` | 网格行数 |
| **方法** | | |
| `__init__` | `(self, grid_cols: int, grid_rows: int) -> None` | 保存网格尺寸，position 初始化为 `(-1, -1)` |
| `respawn()` | `(self, occupied_cells: Iterable[Tuple[int, int]]) -> bool` | 构建所有网格坐标集合 `all_cells`，计算 `available = all_cells - set(occupied_cells)`。若 `available` 为空，返回 `False`（触发胜利）；否则从中 `random.choice` 选取一个赋给 `position`，返回 `True` |
| `reset()` | `(self) -> None` | `position` 重置为 `(-1, -1)`，由下一次 `respawn()` 设置有效位置 |

**`respawn` 算法复杂度**: O(cols * rows) = O(1200)，在每吃到食物时执行一次，完全可接受。

### 1.4 Renderer (`renderer.py`)

**定位**: 封装所有 Pygame 绘制调用，依赖 `pygame`、`Config`。接收 `pygame.Surface` (screen) 作为绘制目标。

```python
class Renderer:
```

| 成员 | 签名 | 说明 |
|------|------|------|
| **属性** | | |
| `screen` | `pygame.Surface` | 游戏窗口 surface，构造时注入 |
| `font_small` | `pygame.font.Font` | HUD 分数字体（字号 24） |
| `font_large` | `pygame.font.Font` | 游戏结束/胜利文字字体（字号 48） |
| `clock` | `pygame.time.Clock` | 帧率控制时钟 |
| **方法** | | |
| `__init__` | `(self, screen: pygame.Surface) -> None` | 保存 screen 引用；初始化两个 Font 对象（使用系统默认字体）；创建 Clock 实例 |
| `draw_background()` | `(self) -> None` | `screen.fill(COLORS.BACKGROUND)` 全局填充背景色；绘制网格线辅助视觉定位（可选，遍历行列画细线） |
| `draw_snake()` | `(self, snake: Snake) -> None` | 遍历 `snake.body[1:]`（蛇身，索引 1 到末尾），每个绘制 `pygame.draw.rect` 矩形（`x*CELL_SIZE, HUD_HEIGHT + y*CELL_SIZE, CELL_SIZE-1, CELL_SIZE-1`，减 1 留间隔让每节更分明）；蛇头 `snake.body[0]` 使用 `COLORS.SNAKE_HEAD` 颜色绘制 |
| `draw_food()` | `(self, food: Food) -> None` | 以 `COLORS.FOOD` 在 `food.position` 对应像素位置绘制矩形或小圆形 |
| `draw_hud()` | `(self, score: int) -> None` | 在窗口顶部 `(10, 5)` 位置以 `font_small` 渲染 `f"Score: {score}"` 文字 |
| `draw_game_over()` | `(self, score: int) -> None` | 绘制半透明遮罩层，居中显示 `"GAME OVER"` 文字和最终得分 `f"Final Score: {score}"`，底部显示 `"Press R to Restart"` 或 `"Press Q to Quit"` 提示 |
| `draw_victory()` | `(self, score: int) -> None` | 类似 game_over，文字改为 `"YOU WIN!"` |
| `draw_frame()` | `(self, snake: Snake, food: Food, score: int, state: GameState) -> None` | 按顺序调用：`draw_background()` -> `draw_snake()` -> `draw_food()` -> `draw_hud()`；最后根据 `state` 值条件调用 `draw_game_over()` 或 `draw_victory()` |
| `tick()` | `(self) -> int` | 委托 `self.clock.tick(FPS)`，返回实际帧间隔 ms |

**坐标映射公式**：网格坐标 `(gx, gy)` 映射到像素：
```
px = gx * CELL_SIZE
py = HUD_HEIGHT + gy * CELL_SIZE
```
`HUD_HEIGHT` 在窗口顶部预留 40px 用于分数显示，游戏区域实际从 y=40 开始。

### 1.5 InputHandler (`input_handler.py`)

**定位**: 处理 Pygame 事件队列，产出统一命令字典。

```python
class InputHandler:
```

| 成员 | 签名 | 说明 |
|------|------|------|
| **属性** | | |
| `_paused` | `bool` | 窗口焦点丢失时的暂停标志 |
| **方法** | | |
| `__init__` | `(self) -> None` | `_paused = False` |
| `process_events()` | `(self, current_direction: Tuple[int, int], state: GameState) -> dict` | 遍历 `pygame.event.get()` 返回的事件列表，生成命令字典。详见下方数据流 |
| `is_paused()` | `(self) -> bool` | 返回 `_paused` |

**`process_events` 返回字典结构**：
```python
{
    "action": "direction" | "restart" | "quit" | "pause" | "resume" | "none",
    "direction": Optional[Tuple[int, int]]  # 仅 action="direction" 时有值
}
```

**`process_events` 内部逻辑**：

```
遍历 pygame.event.get() 事件队列:
  - pygame.QUIT          → 直接返回 {"action": "quit"}
  - pygame.WINDOWEVENT_FOCUS_LOST
      → _paused = True，返回 {"action": "pause"}
  - pygame.WINDOWEVENT_FOCUS_GAINED
      → _paused = False，返回 {"action": "resume"}
  - pygame.KEYDOWN + K_UP    → 若 current_direction != (0,  1): buffered_dir = ( 0, -1)
  - pygame.KEYDOWN + K_DOWN  → 若 current_direction != (0, -1): buffered_dir = ( 0,  1)
  - pygame.KEYDOWN + K_LEFT  → 若 current_direction != (1,  0): buffered_dir = (-1,  0)
  - pygame.KEYDOWN + K_RIGHT → 若 current_direction != (-1, 0): buffered_dir = ( 1,  0)
  - pygame.KEYDOWN + K_r     → 仅当 state in (GAME_OVER, VICTORY) 时，返回 {"action": "restart"}
  - pygame.KEYDOWN + K_q     → 仅当 state in (GAME_OVER, VICTORY) 时，返回 {"action": "quit"}

遍历结束后，若 buffered_dir 不为 None:
  返回 {"action": "direction", "direction": buffered_dir}
否则:
  返回 {"action": "none"}
```

**关键设计**: 方向键不立即生效，而是存入 `buffered_dir` 变量。同一帧内多次方向键按下时，后面的覆盖前面的（取最后一个有效方向）。遍历结束后才返回，实现"每帧单方向缓存"策略。

### 1.6 Game (`game.py`)

**定位**: 游戏协调器。持有所有组件实例，管理状态机，编排主循环。

```python
from enum import Enum, auto

class GameState(Enum):
    RUNNING = auto()     # 游戏进行中
    GAME_OVER = auto()   # 撞墙或撞自身
    VICTORY = auto()     # 填满所有格子（胜利）

class Game:
```

| 成员 | 签名 | 说明 |
|------|------|------|
| **属性** | | |
| `snake` | `Snake` | 蛇实例 |
| `food` | `Food` | 食物实例 |
| `renderer` | `Renderer` | 渲染器实例 |
| `input_handler` | `InputHandler` | 输入处理器实例 |
| `score` | `int` | 当前分数，初始 0 |
| `state` | `GameState` | 当前游戏状态，初始 `RUNNING` |
| `screen` | `pygame.Surface` | 窗口 surface |
| **方法** | | |
| `__init__` | `(self) -> None` | 创建 Pygame 窗口（`pygame.display.set_mode`）；设置窗口标题；实例化 `Snake`、`Food`、`Renderer`、`InputHandler`；调用 `food.respawn(snake.body)` 生成首个食物 |
| `run()` | `(self) -> None` | 主循环入口。```while True: self.renderer.tick(); command = self.input_handler.process_events(self.snake.direction, self.state); self._handle_command(command); self._update(); self.renderer.draw_frame(self.snake, self.food, self.score, self.state); pygame.display.flip()``` |
| `_handle_command()` | `(self, command: dict) -> None` | 根据 `command["action"]` 分发：`"direction"` -> `self.snake.change_direction()`; `"restart"` -> `self.reset()`; `"quit"` -> `pygame.quit(); sys.exit()`; `"pause"` -> 设置暂停标志; `"resume"` -> 清除暂停标志; `"none"` -> 无操作 |
| `_update()` | `(self) -> None` | 仅当 `state == RUNNING` 且未暂停时执行：<br>1. 判定食物碰撞：`grow_flag = (snake.head == food.position)`<br>2. 执行 `snake.move_and_grow(grow_flag)`<br>3. 若 `grow_flag`: `score += SCORE_PER_FOOD`；调用 `food.respawn(snake.body)`；若返回 `False`，`state = VICTORY`<br>4. 若 `snake.check_boundary_collision(GRID_COLS, GRID_ROWS)` 或 `snake.check_self_collision()`: `state = GAME_OVER` |
| `reset()` | `(self) -> None` | `snake.reset()`; `food.reset()`; `food.respawn(snake.body)`; `score = 0`; `state = RUNNING` |

**关键设计说明**：`_update` 中先判定是否吃到食物（`grow_flag`），再执行 `move_and_grow`。这确保蛇头移动到食物位置时，该帧蛇身就增长（新蛇头与食物同位置，身体一节弹出前追加一节）。碰撞检测在移动之后进行，蛇头新位置若出界或碰到身体，下一帧状态切换。

### 3.3 GameState（游戏状态 - 枚举）

| 枚举值 | 含义 |
|--------|------|
| `RUNNING` | 游戏正常运行，接受输入，蛇移动 |
| `GAME_OVER` | 撞墙或撞自身，蛇停止，等待重新开始 |
| `VICTORY` | 蛇身占满所有格子，玩家胜利，等待重新开始 |

状态转换图：

```
    启动 → RUNNING
    RUNNING + 撞墙/自撞 → GAME_OVER
    RUNNING + food.respawn()=False → VICTORY
    GAME_OVER + K_r → RUNNING (reset)
    VICTORY + K_r → RUNNING (reset)
    任意状态 + QUIT → 进程退出
```

### 3.4 Command（输入命令 - 字典结构）

| 字段 | Python 类型 | 必填 | 说明 |
|------|-------------|------|------|
| `action` | `str` | 是 | 取值: `"direction"`, `"restart"`, `"quit"`, `"pause"`, `"resume"`, `"none"` |
| `direction` | `Optional[Tuple[int, int]]` | 否 | 仅 `action="direction"` 时携带方向向量 |

## 4. 关键流程

### 4.1 游戏启动流程

```
main()
  │
  ├─ [1] try: import pygame
  │       except ImportError: print 提示, sys.exit(1)
  │
  ├─ [2] pygame.init()              ← 初始化 Pygame 各子系统
  │
  ├─ [3] Game.__init__()
  │       ├─ pygame.display.set_mode((800, 600))
  │       ├─ pygame.display.set_caption("Snake Game")
  │       ├─ Snake(grid_cols=40, grid_rows=30, initial_length=3)
  │       │     body = [(18,15), (19,15), (20,15)]   ← 水平中心，蛇头在右
  │       │     direction = (1, 0)                     ← 默认向右
  │       ├─ Food(grid_cols=40, grid_rows=30)
  │       │     position = (-1, -1)
  │       ├─ food.respawn(snake.body)
  │       │     构建可用格子集合 → 随机选取 → position 赋值
  │       ├─ Renderer(screen)
  │       │     创建 font_small(24), font_large(48), Clock
  │       └─ InputHandler()
  │
  └─ [4] game.run()                 ← 进入主循环
```

### 4.2 主循环单帧流程

```
Game.run() 每帧:

  ┌─ [step 1] renderer.tick()                         ← clock.tick(FPS) 控制帧率
  │           返回: 实际帧间隔 ms
  │
  ├─ [step 2] events = pygame.event.get()              ← 获取事件队列
  │           command = input_handler.process_events(current_dir, state)
  │
  │   内部逻辑:
  │     - 遍历 events，QUIT → return {"action": "quit"}
  │     - KEYDOWN + 方向键 → buffered_dir = 对应方向 (反向检查不过则跳过)
  │     - KEYDOWN + K_r (GAME_OVER/VICTORY 时) → return {"action": "restart"}
  │     - 遍历结束 → 若有buffered_dir: return {"action":"direction","direction":buffered_dir}
  │                  否则:             return {"action": "none"}
  │
  ├─ [step 3] _handle_command(command)
  │     ├─ "quit"     → pygame.quit(); sys.exit()
  │     ├─ "restart"  → self.reset()
  │     └─ "direction"→ snake.change_direction(dx, dy)  # 反向检查在内部
  │
  ├─ [step 4] _update()
  │   IF state == RUNNING AND NOT paused:
  │     ├─ grow_flag = (snake.head == food.position)
  │     ├─ snake.move_and_grow(grow_flag)
  │     │     body.insert(0, new_head)
  │     │     if not grow_flag: body.pop()
  │     │
  │     ├─ IF grow_flag:
  │     │     score += SCORE_PER_FOOD          # +10
  │     │     ok = food.respawn(snake.body)    # 重新生成食物
  │     │     IF not ok:
  │     │         state = VICTORY             # 满屏胜利
  │     │
  │     ├─ IF snake.check_boundary_collision(cols, rows):
  │     │     state = GAME_OVER
  │     │
  │     └─ IF snake.check_self_collision():
  │           state = GAME_OVER
  │
  ├─ [step 5] renderer.draw_frame(snake, food, score, state)
  │     ├─ draw_background()     → screen.fill(BACKGROUND), 绘制网格线
  │     ├─ draw_snake()          → 从尾到头绘制矩形，蛇头用 HEAD 颜色
  │     ├─ draw_food()           → 在 food.position 绘制
  │     ├─ draw_hud(score)       → 顶部 "Score: {score}"
  │     └─ IF state == GAME_OVER → draw_game_over(score)
  │        IF state == VICTORY   → draw_victory(score)
  │
  └─ [step 6] pygame.display.flip()                    ← 双缓冲交换
```

### 4.3 蛇移动与碰撞检测详细流程

```
snake.move_and_grow(grow_flag=True):

  移动前状态: body = [(5,10), (4,10), (3,10)], direction = (1,0)
  新蛇头: new_head = (5+1, 10+0) = (6, 10)

  步骤:
  1. body.insert(0, (6, 10))
     body = [(6,10), (5,10), (4,10), (3,10)]   ← 长度 +1

  2. if not grow_flag:
       body.pop()
       body = [(6,10), (5,10), (4,10)]         ← 长度恢复

  3. 调用者随后做碰撞检测:
     check_boundary_collision(40, 30):
       head = (6,10)
       6 < 0? No. 6 >= 40? No. 10 < 0? No. 10 >= 30? No.
       → return False

     check_self_collision():
       head = (6,10)
       (6,10) in [(5,10), (4,10), (3,10)]?
       → return False

碰撞示意:

  撞墙:
    new_head = (-1, 10) → check: -1 < 0 → True → GAME_OVER

  自撞:
    body = [(5,10), (6,10), (5,9), (5,10)]
    移动后: body = [(5,11), (5,10), (6,10), (5,9)]
    head = (5,11) in [(5,10), (6,10), (5,9)]? No → OK
    若转弯: body = [(5,10), (4,10), (5,10)]?
    head = (5,10) in [(4,10), (5,10)]? Yes → GAME_OVER
```

### 4.4 食物碰撞与重新生成流程

```
当前帧状态:
  snake.head = (7, 12)
  food.position = (7, 12)      ← 蛇头与食物在同一格
  snake.body = [(7,12), (6,12), (5,12)]

_update():
  grow_flag = True              ← head == food.position
  snake.move_and_grow(True)
    body = [(8,12), (7,12), (6,12), (5,12)]   ← 增长一节，旧食物位置被蛇头占据

  score = 0 + 10 = 10

  food.respawn(snake.body):
    占用集 = {(8,12), (7,12), (6,12), (5,12)}
    全集 = {(0,0)...(39,29)}  共 1200 格
    可用 = 全集 - 占用集 = 1196 格
    position = random.choice(available)      ← 如 (20, 8)
    return True
```

### 4.5 游戏重新开始流程

```
用户在 GAME_OVER 或 VICTORY 状态按下 R 键:

InputHandler.process_events:
  KEYDOWN + K_r, state in (GAME_OVER, VICTORY)
  → return {"action": "restart"}

Game._handle_command({"action": "restart"}):
  → self.reset()

Game.reset():
  snake.reset()
    body = [(18,15), (19,15), (20,15)]
    direction = (1, 0)
  food.reset()
    position = (-1, -1)
  food.respawn(snake.body)     → 随机新位置
  score = 0
  state = RUNNING

```

### 4.6 窗口焦点丢失流程

```
用户 Alt+Tab 切换窗口:

Pygame 事件: WINDOWEVENT_FOCUS_LOST

InputHandler.process_events:
  → _paused = True
  → return {"action": "pause"}

Game._handle_command → 设置暂停标志 (self._paused = True)

Game._update() → 检测到 self._paused，跳过逻辑更新
  蛇停止移动，但渲染继续（画面冻结在当前帧）

用户切回窗口:
Pygame 事件: WINDOWEVENT_FOCUS_GAINED
  → _paused = False
  → return {"action": "resume"}

游戏恢复正常更新
```

## 5. 错误处理

### 5.1 错误码定义

本项目为桌面游戏，不使用 HTTP 状态码，定义内部错误处理策略。

| 错误场景 | 处理方式 | 用户可见行为 |
|----------|----------|-------------|
| **E-009: Pygame 未安装** | `try: import pygame` 在 `main()` 中捕获 `ImportError`，输出提示到 stderr，`sys.exit(1)` | 终端显示 `"Pygame is not installed. Please run: pip install pygame"`，进程退出码 1 |
| **E-001: 食物生成失败（满屏胜利）** | `food.respawn()` 返回 `False`，Game 层设置 `state = VICTORY` | 画面显示 "YOU WIN!" 和最终得分 |
| **E-007: 食物生成落在蛇身位置** | 算法层面保证不可发生：`respawn()` 排除所有 `occupied_cells` | 无此类错误场景 |
| **E-008: 重试兜底** | 使用集合差运算一次性获取所有可用格子，无需重试循环 | 无无限循环风险 |
| **方向键映射错误** | `change_direction` 拒绝非单位向量方向；`process_events` 仅返回 `(1,0), (-1,0), (0,-1), (0,1)` 之一 | 非法按键被忽略 |
| **pygame.init() 失败** | 未捕获，依赖 Pygame 自身的异常传播（通常是 SDL 初始化失败） | Pygame 抛出 `pygame.error` 异常，包含平台相关错误信息 |
| **窗口创建失败** | `pygame.display.set_mode()` 失败时 Pygame 抛异常 | 异常传播，进程退出 |
| **字体加载失败** | 使用 `pygame.font.Font(None, size)` 获取默认系统字体，不会因缺失字体文件而失败 | 使用系统 fallback 字体 |

### 5.2 日志策略

- **不使用日志框架**（如 `logging`）：项目规模小，不引入额外依赖。
- **启动阶段错误**：直接 `print()` 到 stderr + `sys.exit()`。
- **运行时异常**：Pygame 自身异常通过 Python 默认 traceback 输出到 stderr。
- **调试信息**（开发阶段可选）：`print` 输出帧率、蛇位置、状态转换，正式发布前移除。

### 5.3 降级策略

| 场景 | 预期行为 | 降级行为 |
|------|----------|----------|
| 焦点丢失检测 | `WINDOWEVENT_FOCUS_LOST` 触发暂停 | 若平台不支持焦点事件，不暂停，游戏继续运行（E-005） |
| 字体大小不适配 | 使用字号 24 / 48 | 若默认字体不可用，Pygame 自动 fallback |
| 显示模式不支持 | 请求 800x600 窗口 | Pygame/SDL2 自动选择最近的可用模式 |
| 时钟精度不足 | `clock.tick(10)` 控制帧率 | 若系统时钟精度低，实际帧率可能略偏离 10 FPS（仍在 ±20% 可接受范围） |

### 5.4 防御性编程检查点

| 检查点 | 位置 | 处理 |
|--------|------|------|
| `body` 列表中坐标合法性 | `Snake.__init__`, `Snake.reset` | 确保初始坐标在 [0, cols) x [0, rows) 范围内 |
| `direction` 非零向量 | `Snake.change_direction` | 拒绝 `(0,0)` 方向。实际由 `process_events` 保证不会传入 `(0,0)` |
| `food.respawn` 返回值 | `Game._update` | 必须检查布尔返回值，`False` 时切换 VICTORY |
| `body` 不为空 | `Snake.move_and_grow` | `body` 不可能为空（最小长度 1），但可加断言 |
| 食物初始位置 `(-1,-1)` 不应被绘制 | `Renderer.draw_food` | 绘制前检查 `position != (-1, -1)` 避免在无效位置绘制 |
