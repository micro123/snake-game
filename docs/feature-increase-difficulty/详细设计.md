# 详细设计

> **功能**: 分数递增难度 (Score-Based Difficulty Scaling) | **阶段**: 详细设计 (P3) | **角色**: 高级开发工程师 | **日期**: 2026-07-10

## 1. 模块设计

### 1.1 config.py — DifficultyConfig 常量块

| 常量 | 类型 | 默认值 | 职责 |
|------|------|--------|------|
| `SCORE_THRESHOLD_INTERVAL` | `int` | `50` | 每多少分提升一级难度 |
| `DIFFICULTY_INCREMENT` | `float` | `0.1` | 每级难度增加的倍率值 |
| `MIN_TICK_INTERVAL` | `int` | `20` | tick 间隔硬下限（ms），从原 `_get_current_tick_interval()` 中提取为显式常量 |
| `MAX_DIFFICULTY_MULTIPLIER` | `float` | `4.0` | 难度倍率上限，由 `BASE_TICK_INTERVAL / MIN_TICK_INTERVAL - 1` 推导 |

**新增函数 `_validate_difficulty_config()`**（由 `_validate_boost_config()` 末尾调用）：
- 校验 `SCORE_THRESHOLD_INTERVAL >= 1`，否则重置为 `50`
- 校验 `DIFFICULTY_INCREMENT >= 0.0`，否则重置为 `0.1`
- 校验 `MIN_TICK_INTERVAL` 在 `(0, BASE_TICK_INTERVAL]` 范围内，否则重置为 `20`
- 推导并赋值 `MAX_DIFFICULTY_MULTIPLIER = (BASE_TICK_INTERVAL / MIN_TICK_INTERVAL) - 1.0`，保证叠加后不会跌破下限
- 所有越界均输出 `logging.WARNING` 并自动修正

### 1.2 Game — DifficultyState 状态管理

| 类 | 方法签名 | 属性 | 职责 |
|----|---------|------|------|
| `Game` | `__init__(self) -> None` | `difficulty_level: int = 0`<br>`difficulty_multiplier: float = 0.0` | 初始化时新增两个难度状态属性，均为零值（对应初始速度 1.0x） |
| `Game` | `_update_difficulty(self) -> None` | — | 根据 `self.score` 重新计算 `difficulty_level` 和 `difficulty_multiplier`。幂等公式：`level = score // SCORE_THRESHOLD_INTERVAL`，`multiplier = min(level * DIFFICULTY_INCREMENT, MAX_DIFFICULTY_MULTIPLIER)` |
| `Game` | `_get_current_tick_interval(self) -> float` | — | **核心修改**：公式从 `BASE / multiplier` 改为 `BASE / (1.0 + difficulty_multiplier + (boost_multiplier - 1.0))`，下限改用 `MIN_TICK_INTERVAL` 常量替代硬编码 20.0。保留 NaN/ZeroDivisionError 的 try/except 保护 |
| `Game` | `_update(self) -> None` | — | 在 `self.score += SCORE_PER_FOOD` 之后、`self.food.respawn()` 之前插入 `self._update_difficulty()` 调用 |
| `Game` | `reset(self) -> None` | — | 在 `self.score = 0` 之后追加 `self.difficulty_level = 0` 和 `self.difficulty_multiplier = 0.0`，确保每局难度从零开始 |
| `Game` | `run(self) -> None` | — | `renderer.draw_frame()` 调用点新增 `difficulty_level` 和 `effective_multiplier` 两个参数 |
| `Game` | (无新增方法) | `effective_multiplier: float`（局部变量，在 `run()` 中计算） | `effective = 1.0 + self.difficulty_multiplier + (self.snake.boost_multiplier - 1.0)`，每次渲染帧前计算，传递给 Renderer |

### 1.3 Snake — 无变更

Snake 模块不涉及任何修改。`boost_multiplier` 属性和 `boost_state` 字典行为不变，`difficulty_multiplier` 的叠加由 Game 在 `_get_current_tick_interval()` 中完成。

### 1.4 Renderer — HUD 扩展

| 类 | 方法签名 | 属性 | 职责 |
|----|---------|------|------|
| `Renderer` | `draw_hud(self, score: int, is_boosting: bool = False, difficulty_level: int = 0, effective_multiplier: float = 1.0) -> None` | — | **签名扩展**：新增 `difficulty_level` 和 `effective_multiplier` 参数。在 HUD 背景条上新增一行居中文字 `Lv.{difficulty_level}  Speed {effective_multiplier:.1f}x`。使用现有 `self.font_small` 字体，不创建新 Font 对象。当 `difficulty_level == 0` 且 `effective_multiplier == 1.0` 时仍渲染（保持布局一致，显示 `Lv.0  Speed 1.0x`） |
| `Renderer` | `draw_frame(self, snake: Snake, food: Food, score: int, state: GameState, is_boosting: bool = False, difficulty_level: int = 0, effective_multiplier: float = 1.0) -> None` | — | **签名扩展**：新增 `difficulty_level` 和 `effective_multiplier` 参数，直接透传给 `draw_hud()` |

**HUD 文字布局**（在现有 40px 高度内）：
- 左对齐：`Score: {score}`（位置不变）
- 居中对齐：`Lv.{difficulty_level}  Speed {effective_multiplier:.1f}x`（新增）
- 右对齐：`BOOST`（位置不变，条件渲染）

使用 `get_rect()` 计算 `midtop` / `center` 定位，颜色均为 `COLORS.TEXT`（白色），与现有 HUD 风格一致。

## 3. 数据模型

### 3.1 Game（扩展属性 — 难度状态）

| 属性 | 类型 | 初始值 | 约束 | 描述 |
|------|------|--------|------|------|
| `difficulty_level` | `int` | `0` | `>= 0`，由 `score // SCORE_THRESHOLD_INTERVAL` 计算 | 当前难度等级编号，驱动 difficulty_multiplier |
| `difficulty_multiplier` | `float` | `0.0` | `[0.0, MAX_DIFFICULTY_MULTIPLIER]`，由 `min(level * DIFFICULTY_INCREMENT, MAX_DIFFICULTY_MULTIPLIER)` 计算 | 难度对速度的倍率贡献（超出基础速度的部分） |
| `effective_multiplier`（run 局部变量） | `float` | `1.0` | `>= 1.0`，`1.0 + difficulty_multiplier + (boost_multiplier - 1.0)` | 综合速度倍率（渲染 HUD 用），每次帧计算 |

生命周期：与 Game 实例同生命周期，由 `reset()` 清零。

### 3.2 DifficultyConfig（config.py 新增常量）

| 常量 | 类型 | 默认值 | 约束 | 描述 |
|------|------|--------|------|------|
| `SCORE_THRESHOLD_INTERVAL` | `int` | `50` | `>= 1` | 分数每增此值，difficulty_level +1 |
| `DIFFICULTY_INCREMENT` | `float` | `0.1` | `>= 0.0` | 每级 difficulty_multiplier 增加此值 |
| `MIN_TICK_INTERVAL` | `int` | `20` | `0 < MIN_TICK_INTERVAL <= BASE_TICK_INTERVAL` | tick 间隔最小毫秒数（硬下限） |
| `MAX_DIFFICULTY_MULTIPLIER` | `float` | `4.0` | 自动推导，=`BASE_TICK_INTERVAL / MIN_TICK_INTERVAL - 1` | 难度倍率上限，与 MAX_BOOST_MULTIPLIER 叠加后保证 tick >= MIN_TICK_INTERVAL |

**关系**：
- `MAX_DIFFICULTY_MULTIPLIER` 由 `BASE_TICK_INTERVAL` 和 `MIN_TICK_INTERVAL` 推导，不可直接配置
- `difficulty_level * DIFFICULTY_INCREMENT` 经 `min()` 钳位至 `MAX_DIFFICULTY_MULTIPLIER`
- `1.0 + difficulty_multiplier + (boost_multiplier - 1.0)` 作为最终除数，MAX_DIFFICULTY_MULTIPLIER + MAX_BOOST_MULTIPLIER 可能超出可用于除数，但 `max(interval, MIN_TICK_INTERVAL)` 始终兜底

### 3.3 实体关系

```
config.py: DifficultyConfig 常量
    │
    │ 导入并引用
    ▼
game.py: Game
    ├── difficulty_level: int         ←── score // SCORE_THRESHOLD_INTERVAL
    ├── difficulty_multiplier: float  ←── level * DIFFICULTY_INCREMENT  (clamped)
    │
    │ 在 _get_current_tick_interval() 中复合计算:
    │   effective = 1.0 + difficulty_multiplier + (snake.boost_multiplier - 1.0)
    │   interval = BASE_TICK_INTERVAL / effective  (下限 = MIN_TICK_INTERVAL)
    │
    └── snake.py: Snake.boost_multiplier (只读属性，返回 >= 1.0)
```

## 4. 关键流程

### 4.1 分数驱动难度递增流程

```
玩家吃食物
  │
  ▼
Game._update()
  ├── grow_flag = (new_head == food.position) → True
  ├── snake.move_and_grow(grow_flag=True)
  ├── self.score += SCORE_PER_FOOD          // score 从 50 → 60
  │
  ├── self._update_difficulty()             // ★ 新增调用
  │   ├── level = 60 // 50 = 1             // difficulty_level: 0→1
  │   ├── raw = 1 * 0.1 = 0.1              // 累加 0.1
  │   └── self.difficulty_multiplier = min(0.1, 4.0) = 0.1
  │
  ├── food.respawn(snake.body)
  └── ...碰撞检测...
```

### 4.2 Tick 间隔计算流程

```
主循环 while accumulator >= tick_interval:
  │
  ▼
Game._get_current_tick_interval()
  │
  ├── boost_extra = self.snake.boost_multiplier - 1.0  // boost 贡献 (0..4)
  ├── effective = 1.0 + self.difficulty_multiplier + boost_extra
  │     // 无 boost 无 diff: 1.0+0+0=1.0  → 100ms
  │     // boost 2x 无 diff:  1.0+0+1.0=2.0 → 50ms
  │     // Lv.5 diff 无 boost: 1.0+0.5+0=1.5 → 66.7ms
  │     // Lv.5 diff + boost:  1.0+0.5+1.0=2.5 → 40ms
  │
  ├── interval = BASE_TICK_INTERVAL / effective
  │     // try/except 保护 NaN / ZeroDivisionError
  │
  └── return max(MIN_TICK_INTERVAL, interval)  // 硬下限 20ms
```

### 4.3 游戏重置流程

```
Game.reset()
  ├── self.snake.reset()               // 蛇身 + boost_state 复位
  ├── self.food.reset()
  ├── self.food.respawn(self.snake.body)
  ├── self.score = 0
  ├── self.state = GameState.RUNNING
  ├── self.difficulty_level = 0        // ★ 新增：难度等级归零
  ├── self.difficulty_multiplier = 0.0 // ★ 新增：难度倍率归零
  └── self._reset_boost()              // boost 双重保险

结果：_get_current_tick_interval() → 1.0 + 0.0 + 0 = 1.0 → 100ms ✓
```

### 4.4 HUD 渲染流程

```
主循环每帧:
  │
  ├── effective = 1.0 + self.difficulty_multiplier + (self.snake.boost_multiplier - 1.0)
  │
  └── renderer.draw_frame(
        snake, food, score, state,
        is_boosting=self.snake.is_boosting,
        difficulty_level=self.difficulty_level,     // ★ 新增
        effective_multiplier=effective,             // ★ 新增
      )
        │
        └── draw_hud(score, is_boosting, difficulty_level, effective_multiplier)
              │
              ├── 背景条 (不变)
              ├── "Score: {score}"          — 左侧
              ├── "Lv.{level}  Speed {m}x" — 居中 ★ 新增
              └── "BOOST"                   — 右侧 (条件)
```

### 4.5 手动加速与难度加速叠加流程

```
用户按住空格
  │
  ▼
主循环: _update_boost_state(dt, boost_pressed=True)
  │ 线性插值: current_multiplier 从 1.0 → 2.0 (150ms 过渡)
  │ snake.boost_state['current_multiplier'] → 2.0
  ▼

主循环: _get_current_tick_interval()
  │ effective = 1.0 + difficulty_multiplier + (2.0 - 1.0)
  │           = 1.0 + difficulty_multiplier + 1.0
  │ 例如 difficulty=0.3: effective = 2.3, interval = 100/2.3 ≈ 43.5ms
  ▼

用户释放空格
  │
  ▼
主循环: _update_boost_state(dt, boost_pressed=False)
  │ 线性插值: current_multiplier 从 2.0 → 1.0
  ▼

主循环: _get_current_tick_interval()
  │ effective = 1.0 + difficulty_multiplier + 0 = 1.0 + difficulty_multiplier
  │ tick 间隔平滑回升，不跳跃
```

## 5. 错误处理

### 5.1 错误码

| 错误码 | HTTP 状态 | 描述 | 可重试 | 处理方式 |
|--------|-----------|------|--------|----------|
| `CFG-004` | N/A（客户端日志） | `SCORE_THRESHOLD_INTERVAL < 1`，配置无效，自动重置为 50 | N/A | `logging.WARNING` + 自动修正 |
| `CFG-005` | N/A（客户端日志） | `DIFFICULTY_INCREMENT < 0`，配置无效，自动重置为 0.1 | N/A | `logging.WARNING` + 自动修正 |
| `CFG-006` | N/A（客户端日志） | `MIN_TICK_INTERVAL` 越界，自动重置为 20 | N/A | `logging.WARNING` + 自动修正 |
| `RT-001` | N/A（客户端日志） | tick_interval 计算为 NaN/0/负值，回退到 BASE_TICK_INTERVAL | 是（下一帧重新计算） | `logging.ERROR` + 回退安全值 |
| — | N/A | `difficulty_multiplier` 浮点精度：使用 `level * increment` 幂等公式，无累加误差 | N/A | 无误差累积，无需处理 |

### 5.2 日志策略

- **配置校验阶段**（模块导入时）：使用 `logging.WARNING` 记录所有配置越界和自动修正行为，日志前缀 `CFG-0XX`
- **运行时 tick 计算**：沿用现有 `logging.ERROR` 级别记录 `RT-001` 异常（tick_interval 计算无效时的回退），日志包含当前 effective_multiplier 值用于排查
- **开发调试**：`_update_difficulty()` 内部不添加日志（每吃食物调用一次，频率低但无排查价值），基础信息可从 `difficulty_level` 和 `score` 推断

### 5.3 降级策略

| 异常场景 | 降级行为 | 影响 |
|----------|----------|------|
| `difficulty_multiplier` 计算出 NaN/Inf（极端场景：`DIFFICULTY_INCREMENT = float('inf')` 经校验阻止，不会发生） | `_get_current_tick_interval()` 的 try/except 捕获所有除零/NaN，回退 `interval = BASE_TICK_INTERVAL` = 100ms | 难度加速暂时失效，游戏以基础速度运行，不崩溃 |
| `effective_multiplier = 0`（需 boost_multiplier < 1.0 且 difficulty_multiplier < -1.0，两者均有边界 guard 阻止） | `max(MIN_TICK_INTERVAL, interval)` 兜底于 20ms | 不会低于下限 |
| `difficulty_level` 溢出为极大整数（分数极高时） | `difficulty_multiplier = min(level * increment, MAX_DIFFICULTY_MULTIPLIER)` 自动钳位 | 难度不会无限加速，tick 间隔不低于 20ms |
| `Renderer.draw_hud()` 未收到新参数（向后兼容调用） | 参数默认值 `difficulty_level=0, effective_multiplier=1.0`，HUD 显示 `Lv.0  Speed 1.0x` | 功能完整，只是数值为默认值 |
| 配置常量被意外改为非法值 | `_validate_difficulty_config()` 在模块导入时自动修正并警告 | 游戏可正常启动，使用修正后的安全默认值 |

### 5.4 防御性编程要点

1. **除零保护**：`_get_current_tick_interval()` 中 `effective_multiplier` 作为除数，try/except 捕获 `ZeroDivisionError`。理论上 `effective_multiplier >= 1.0` 始终成立，此保护为纵深防御
2. **NaN 保护**：`interval != interval` 检查捕获 NaN，回退到安全默认值
3. **下限钳位**：`max(MIN_TICK_INTERVAL, interval)` 作为最后防线
4. **上限钳位**：`min(level * increment, MAX_DIFFICULTY_MULTIPLIER)` 和 `min(MAX_BOOST_MULTIPLIER, current)` 分别限制两个因子
5. **幂等计算**：`difficulty_multiplier` 每次通过 `level * increment` 完整重算，不依赖上一次值（`reset()` 后完全清零）
6. **默认参数向后兼容**：`draw_hud()` 和 `draw_frame()` 的新参数均有默认值，现有调用不会因缺参而失败
