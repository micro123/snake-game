# 需求分析

> **功能**: 经典贪吃蛇游戏 (Snake Game) | **项目**: feature-snake-game | **阶段**: 需求分析 | **角色**: 需求分析师 + 需求工程师 | **日期**: 2026-07-07

---

## 1. 问题陈述

用户需要在本地环境运行一个轻量级、即开即玩的经典贪吃蛇游戏，以消遣娱乐或回顾经典游戏体验。目前仓库为空，缺少一个可运行的游戏程序。该游戏需具备贪吃蛇核心玩法：方向键控制移动、吃食物增长与得分、撞墙或撞自身判定失败，提供即时视觉反馈与分数统计。

---

## 2. 目标用户

希望在本地快速运行贪吃蛇游戏的休闲玩家。用户通过终端启动游戏后，使用键盘方向键操控，无需网络连接或外部账号。用户期待操作响应及时、碰撞判定准确、分数实时可见。

## 4. 整体验收标准

- 游戏窗口能正常启动，窗口标题包含游戏名称，窗口尺寸固定且合理（如 800x600 或 640x480）。
- 蛇在游戏开始时以固定长度（如3节）出现在画面中，默认向右移动。
- 按下方向键（上/下/左/右）后蛇的移动方向立即改变，且不能反向（如向右时不能直接左转）。
- 食物以明显区别于背景和蛇的颜色显示在游戏区域内的随机空白位置。
- 蛇头碰到食物时，蛇身增长一节，分数增加固定值（如10分），食物立即在另一空白位置重新生成。
- 蛇头碰到游戏区域边界时，游戏停止，显示游戏结束信息。
- 蛇头碰到蛇身任意一节时，游戏停止，显示游戏结束信息。
- 游戏运行期间帧率稳定，无卡顿或明显延迟，输入响应延迟在可接受范围内。
- 分数在游戏界面上持续可见并实时更新。

- 假设用户已安装 Python 3.x 和 Pygame 库，或愿意按照安装说明自行安装

- 假设食物不会生成在蛇身占据的格子上
- 假设同时只存在一个食物
- 假设使用 Windows/Linux/macOS 桌面环境运行，不做平台特殊适配

## 7. 功能性需求

### FR-001 键盘方向键控制蛇的移动方向
- **描述**：玩家通过键盘方向键（上/下/左/右）控制蛇头移动方向。蛇持续沿当前方向移动，方向键按下后立即切换至对应方向。不允许反向移动（如当前向右移动时，按下左键应被忽略）。
- **优先级**：P0
- **验收标准**：
  - 按下上/下/左/右方向键后，蛇头方向在下一逻辑帧立即切换
  - 当前方向的反方向输入被忽略，蛇方向不改变
  - 非方向键按键不产生任何效果
  - 游戏启动时蛇默认向右移动

### FR-002 蛇吃食物后增长与得分
- **描述**：蛇头移动至食物所在格子时判定为"吃到食物"。蛇身增长 1 节（新节追加至蛇尾），分数增加 10 分。食物随即消失并在新位置生成。
- **优先级**：P0
- **验收标准**：
  - 蛇头坐标与食物坐标重合时，蛇身节数 +1，分数 +10
  - 新增长的身体节出现在蛇尾末端
  - 分数变化在界面上即时反映
  - 食物在蛇头碰到后立即移除，不会残留

### FR-003 撞墙判定游戏结束
- **描述**：蛇头移动至游戏区域边界之外（即蛇头坐标超出有效格子范围）时，判定为撞墙，游戏立即停止。
- **优先级**：P0
- **验收标准**：
  - 蛇头坐标超出 [0, 列数-1] 或 [0, 行数-1] 范围时，游戏停止
  - 游戏停止后蛇不再移动
  - 界面显示游戏结束提示信息

### FR-004 撞自身判定游戏结束
- **描述**：蛇头移动至与蛇身任意一节重合的格子时，判定为撞自身，游戏立即停止。
- **优先级**：P0
- **验收标准**：
  - 蛇头坐标与蛇身列表中除蛇头外的任一节的坐标重合时，游戏停止
  - 注意：蛇移动逻辑应先移动蛇头、再检查碰撞。应区分"转向导致撞自身"与"移动前已重合"的场景
  - 游戏停止后蛇不再移动
  - 界面显示游戏结束提示信息

### FR-005 实时分数显示
- **描述**：游戏界面持续显示当前得分，分数变化时即时更新显示。
- **优先级**：P0
- **验收标准**：
  - 分数在游戏窗口固定区域显示（如左上角或顶部状态栏）
  - 游戏开始时分数为 0
  - 吃到食物后分数立即更新显示
  - 字体清晰可辨，与背景有足够对比度

### FR-006 食物随机生成
- **描述**：游戏区域内随机选择一个未被蛇身占据的空白格子生成食物。食物以明显区别于背景和蛇的颜色呈现。
- **优先级**：P0
- **验收标准**：
  - 游戏启动时自动生成第一个食物
  - 食物坐标随机，且不落在蛇身任意一节占据的格子上
  - 同一时间仅存在一个食物
  - 食物颜色与背景色、蛇身颜色有明显区分度
  - 当所有格子被蛇身占满（理论上蛇长 = 总格数）时，视为玩家胜利

### FR-007 游戏结束显示与重新开始
- **描述**：游戏结束后，界面显示最终得分，并提供重新开始游戏的方式。
- **优先级**：P1
- **验收标准**：
  - 游戏结束时显示最终得分和"游戏结束"文字
  - 提供重新开始的操作方式（如按 R 键或空格键重新开始）
  - 重新开始后，蛇恢复初始长度和初始位置，分数归零，食物重新随机生成

### FR-008 游戏主循环与帧率控制
- **描述**：游戏以固定逻辑帧率运行主循环（初始化 -> 事件处理 -> 状态更新 -> 渲染 -> 帧率控制），确保画面渲染流畅。
- **优先级**：P0
- **验收标准**：
  - 游戏窗口正常启动，窗口标题包含游戏名称
  - 逻辑更新帧率固定（如每秒 10 帧），不随游戏进度变化
  - 渲染帧率不低于逻辑更新帧率，无画面撕裂或明显卡顿
  - 窗口尺寸固定

### FR-009 蛇初始状态
- **描述**：游戏启动时蛇以固定长度出现在游戏区域中，默认向右移动。蛇身水平排列连续。
- **优先级**：P0
- **验收标准**：
  - 蛇初始长度为 3 节
  - 蛇头位于蛇身最右端，默认向右移动
  - 蛇身初始位于游戏区域内的有效位置（不含边界外）
  - 蛇身各节之间连续、无间隙

## 8. 非功能性需求

### 8.1 性能
- 逻辑更新帧率稳定在固定值（如 10 FPS），帧间隔偏差不超过 20%
- 输入响应延迟不超过 2 个逻辑帧（即 200ms 内按键效果可见）
- 渲染帧率保持在 30 FPS 或以上，画面流畅无卡顿
- 游戏启动时间不超过 2 秒
- 内存占用稳定，无持续增长（无内存泄漏）

### 8.2 安全
- 游戏为纯本地运行，不涉及网络通信，不访问文件系统（除加载依赖库外）
- 不收集任何用户数据，无隐私风险
- 键盘输入仅影响游戏逻辑，不触发系统级操作

### 8.3 可用性
- 游戏窗口大小固定（如 800x600 或 640x480 像素），打开即玩，无需配置
- 方向键为直觉操作方式，无需额外学习
- 蛇、食物、背景使用高对比度颜色，视觉区分明确
- 分数文字大小适中，处于游戏窗口内可见位置
- 游戏结束信息醒目，重新开始操作简单明确（单键触发）
- 游戏窗口应支持标准窗口操作（关闭按钮正常退出）

### 8.4 可靠性
- 碰撞检测精确到网格单元级别，无误判、漏判
- 食物不会生成在蛇身占据的格子上（100% 保证）
- 蛇移动方向切换逻辑正确，反向输入被正确处理（忽略而非崩溃）
- 游戏结束时状态一致：蛇停止移动、分数定格、界面更新完毕
- 窗口关闭时游戏进程正常退出，无残留进程

## 9. 约束条件

- **运行环境**：需安装 Python 3.x 解释器与 Pygame 库。不提供编译后的可执行文件或安装包。
- **平台**：支持 Windows、Linux、macOS 桌面环境（依赖 Pygame 跨平台能力），不做平台特殊适配。
- **窗口尺寸**：固定尺寸（如 800x600 或 640x480 像素），不可调整大小。
- **网格系统**：蛇沿网格移动，网格单元大小固定（如 20x20 像素）。游戏区域由固定行列数（如 40x30 或 32x24）构成。
- **游戏速度**：逻辑更新帧率固定（如 10 FPS），不随分数增长而加速。
- **食物规则**：同时仅存在一个食物。
- **不可反向**：蛇不能直接反向移动（如向右移动时不能直接左转）。
- **排除范围**：不包含网络联机、多人对战、关卡系统、难度选择、音效、背景音乐、最高分持久化存储、暂停功能、移动端/Web 端适配、道具系统。

## 11. 数据实体

### 11.1 Snake（蛇）
| 字段 | 类型 | 描述 |
|------|------|------|
| body | List[(x, y)] | 蛇身坐标列表，索引 0 为蛇头，末尾为蛇尾 |
| direction | (dx, dy) | 当前移动方向向量，如 (1, 0) 表示向右 |
| length | int | 蛇身长度（等于 body 列表长度） |

**关系**：Snake 与 Food 通过碰撞检测关联；Snake 与 GameBoard 通过边界检测关联。

### 11.2 Food（食物）
| 字段 | 类型 | 描述 |
|------|------|------|
| position | (x, y) | 食物在网格中的坐标 |

**关系**：Food 与 Snake 的 body 互斥（不能共享同一格子）。

### 11.3 GameState（游戏状态）
| 字段 | 类型 | 描述 |
|------|------|------|
| score | int | 当前分数，初始为 0 |
| is_running | bool | 游戏是否正在运行 |
| is_over | bool | 游戏是否已结束 |
| grid_width | int | 网格列数（以格子为单位） |
| grid_height | int | 网格行数（以格子为单位） |
| cell_size | int | 每个格子的像素尺寸 |

### 11.4 GameBoard（游戏区域）
| 字段 | 类型 | 描述 |
|------|------|------|
| width | int | 窗口宽度（像素） |
| height | int | 窗口高度（像素） |
| columns | int | 网格列数 = width / cell_size |
| rows | int | 网格行数 = height / cell_size |

**关系**：GameBoard 定义了 Snake 和 Food 的活动坐标范围。

## 12. 边界情况

| 编号 | 边界情况 | 预期处理 |
|------|----------|----------|
| E-001 | 食物生成时所有可用格子被蛇身占满 | 判定玩家胜利，游戏结束并显示胜利信息 |
| E-002 | 蛇头在边界上方向键指向边界外 | 下一帧蛇头移出边界，触发撞墙游戏结束 |
| E-003 | 蛇长度仅 1 节（理论最小值，非初始状态）时撞自身 | 不存在自身碰撞可能（蛇头不与任何蛇身节重合），正常继续 |
| E-004 | 玩家快速连续按下多个方向键 | 仅最后一个有效方向键在下一逻辑帧生效。反向输入被忽略。处理队列中的中间按键可能导致自撞 |
| E-005 | 窗口失去焦点（如 Alt+Tab 切换） | 游戏暂停逻辑更新或继续运行均可接受，但不应崩溃；推荐暂停以避免失去控制导致意外死亡 |
| E-006 | 关闭窗口按钮被点击 | 游戏进程正常退出，无异常或残留进程 |
| E-007 | 食物随机落在蛇头位置（蛇头刚离开但蛇尾未跟上） | 不应发生：生成食物时必须检查所有蛇身格子（含蛇头），排除所有占据位置 |
| E-008 | 蛇移动到食物位置但食物生成逻辑出错未生成新食物 | 游戏结束后或蛇无法得分，需确保食物生成逻辑有兜底（如重试直到成功） |
| E-009 | Pygame 未安装或 Python 版本不兼容 | 启动时报错，给出清晰的依赖缺失提示（由架构设计决定实现方式） |
