# 方案与评估

> **Bug**: bug-snake-reverse | **阶段**: 方案与评估 (P2) | **角色**: solution-designer + tech-lead | **日期**: 2026-07-10

---

## 1. 可行性判断

**结论: 可行 (yes)**

根因分析定位准确：`process_events()` 和 `change_direction()` 的反向检测基准均使用 `self.direction`（可被立即修改的值），跨帧时方向漂移导致反向拦截被绕过。主方案新增 `committed_direction` 属性仅在 `move_and_grow()`（物理移动）时更新，语义与“蛇实际运动方向”完全对齐，天然消除漂移。改动仅涉及 `snake.py`（新增属性 / 3 处初始化 / `move_and_grow` 更新 / `change_direction` 检测基准）和 `game.py`（1 处传参变更），不改变 `InputHandler` 接口或任何渲染/食物/配置逻辑。`_is_reverse` 静态方法逻辑不变。改动量极小、风险可控、根因消除彻底。

---

## 2. 主方案详述 (Primary Solution)

### 2.1 方案描述

在 `Snake` 中新增 `committed_direction` 属性，仅在 `move_and_grow()` 中更新（记录蛇实际物理移动的方向）。将 `process_events()` 和 `change_direction()` 的反向检测基准从可变的 `self.direction` 改为 `committed_direction`，确保跨帧方向输入的反向比对基准稳定不变，直到蛇实际移动后才更新。

### 2.2 设计依据

仅新增一个属性且改动集中在两个文件，不改动 `InputHandler` 接口语义（仍接收 `direction` 参数）。`committed_direction` 在 `move_and_grow()` 中更新，与蛇物理移动同步，天然解决跨帧方向漂移问题。`change_direction()` 的二次防御同步加固，无功能冗余。改动最小、风险最低，不涉及事件处理或渲染逻辑。

### 2.3 文件变更清单

| 文件 | 变更类型 | 描述 |
|------|----------|------|
| `snake.py` | modify | 新增 `committed_direction` 属性（`__init__`、`_init_body`、`reset` 中赋初值）。`move_and_grow()` 末尾记录 `self.committed_direction = self.direction`。`change_direction()` 的反向检测改为基于 `self.committed_direction` 而非 `self.direction`。 |
| `game.py` | modify | `run()` 中 `process_events()` 调用：将参数从 `self.snake.direction` 改为 `self.snake.committed_direction`。 |

### 2.4 不影响模块

`config.py`, `input_handler.py`, `food.py`, `renderer.py` — 无需任何修改。

## 3. 备选方案分析

### 方案 B：帧级方向缓存

**描述**: 在 `Game.run()` 帧首读取 `self.snake.direction` 存入局部变量，整帧使用该局部变量作为反向检测基准。

| 优点 | 缺点 |
|------|------|
| 改动最少，仅 `game.py` 一行变更 | catchup 多 `_update` 场景下，方向缓存与实际移动可能不一致 |
| 无需修改 `Snake` 类 | `change_direction()` 二次检测仍基于 `self.direction`，两处检测基准不一致 |

**不采纳原因**: catchup 场景下可信度下降，且 `change_direction` 二次防御未加固，存在残留风险。

### 方案 C：方向命令队列

**描述**: `process_events` 产出方向变更入队，`_update` 前出队应用，反向检测用 `last_dequeued` 方向。

| 优点 | 缺点 |
|------|------|
| 架构最清晰，输入与逻辑完全解耦 | 改动量大，涉及 `input_handler`/`game`/`snake` 三模块 |
| 支持未来多方向缓冲策略扩展 | 引入队列管理复杂度，增加内存开销和测试成本 |
| | 与现有 per-frame 缓冲策略需要协调设计 |

**不采纳原因**: 过度设计，对于当前单方向缓冲策略和低复杂度项目，性价比低。

## 4. 回归风险评估

**综合评级: 低 (low)**

回归风险低，原因如下：

1. 仅新增属性，不删除或重构任何现有接口
2. `process_events()` 和 `_is_reverse()` 实现不变，仅调用方传入参数值语义从 `self.direction` 变为 `committed_direction`，类型仍为 `Tuple[int, int]`
3. `change_direction()` 的反向检测基准从 `self.direction` 变为 `self.committed_direction`，语义更严格（检测“是否反转实际移动方向”而非“是否反转当前暂存值”），仅新增拦截不会放宽已有检查
4. `committed_direction` 的初始化路径（`__init__`/`_init_body`/`reset`）均与 `direction` 初始值 `(1, 0)` 一致，首次输入不受影响
5. `move_and_grow()` 已稳定运行，新增一行赋值不改变其控制流

**唯一需关注**: 若 `_init_body` / `reset` 任一处遗漏 `committed_direction` 初始化，将导致 behavior 退化（值为未定义或过时值）。但属实现细节，方案已明确要求三处同步初始化。

## 6. 测试计划

### 6.1 单元测试（新增）

- **TestSnakeCommittedDirection**: 验证 `committed_direction` 初始值与 `direction` 一致
- **TestSnakeCommittedDirection**: `move_and_grow()` 后 `committed_direction` 更新为移动时使用的 `direction`
- **TestSnakeCommittedDirection**: `change_direction` 后 `committed_direction` 不变（直到 `move_and_grow` 执行）
- **TestSnakeCommittedDirection**: `reset()` 后 `committed_direction` 重置为 `(1, 0)`
- **TestCrossFrameReverseIntercept**: 模拟帧 1 `process_events(committed=UP)` 按下 LEFT 被接受后 `change_direction(LEFT)` 但不执行 `_update`；帧 2 `process_events(committed=UP)` 按下 DOWN，验证被拦截返回 `none`
- **TestCrossFrameReverseIntercept**: 帧 1 正常 LEFT -> `_update` 后 `committed=LEFT`；帧 2 `process_events(committed=LEFT)` 按下 DOWN，验证被接受（合法 L 形转弯）
- **TestChangeDirection**: 更新测试中 `change_direction` 使用 `committed_direction` 验证反向拦截正确性

### 6.2 集成测试（新增）

- 端到端：蛇向上移动时快速连按 LEFT+DOWN 两帧，验证 DOWN 被拦截且蛇不反转不死亡
- 端到端：蛇向上移动时按 LEFT -> `_update` 执行 -> 按 DOWN，验证合法 L 形转弯不发生自撞（直线蛇身）
- 回归：现有全部输入处理/蛇实体/游戏逻辑集成测试通过
- 回归：E-004 快速连按多方向键测试通过

### 6.3 手动验证

1. 启动游戏，蛇向右移动，先按上键使蛇向上行进
2. 快速依次按下左键然后立即按下键（模拟跨两帧输入）
3. 确认蛇不向下反转，不触发 Game Over
4. 确认蛇能正常向左转（正交方向不受影响）
5. 确认合法 L 形转弯场景（先左转移动一格，再下转）正常工作

### 6.4 回归检查

- 所有 unittest 套件通过 (`test_snake` / `test_game` / `test_input_handler` / `test_integration` / `test_food` / `test_renderer` / `test_config`)
- 加速功能不受影响（BOOST_KEY 与方向键正交）
- 暂停/恢复功能不受影响
- R 键重启后 `committed_direction` 正确重置

## 7. 测试覆盖缺口

| 缺口 | 描述 |
|------|------|
| 跨帧反向拦截 — 无 `_update` 场景 | 帧 N `process_events(committed=UP)` 接受 LEFT，`_handle_command` 变更 `snake.direction=LEFT` 但 accumulator 未达阈值不触发 `_update`；帧 N+1 `process_events(committed=UP)` 检测 DOWN 应被拦截（committed 未变）。此场景对修复正确性关键，当前测试计划未显式列出。 |
| catchup 多 `_update` 场景 | 单帧内 accumulator 触发 2 次以上 `_update`，方向可能在两次 update 间被 `_handle_command` 修改。需验证 `committed_direction` 在每次 `move_and_grow` 后同步更新，且反向检测始终基于最新的 committed 值。 |
| `committed_direction` 与 `grow_flag` 交互 | `move_and_grow(grow_flag=True)` 后 `committed_direction` 不应有差异。虽属显而易见的正确性，但应作为边界用例覆盖。 |
| `committed_direction` 序列化 / 可观察性 | 当前无 getter，若未来需要序列化或外部观察，需补充属性或方法。现阶段不阻塞。 |

## 10. 最终建议

**结果: approved（批准实施）**

跨帧反向绕过是真实缺陷，根因（反向检测基准随 `direction` 漂移）定位准确。主方案（新增 `committed_direction` 在 `move_and_grow` 时更新）从根因消除问题，改动仅 `snake.py` + `game.py` 共 2 个文件约 8 行代码，不改变接口。回归风险低。预估 2.5 人天工时合理。
