#### T-005 game.py 主循环解耦重构

- **变更文件**: `game.py` (修改), `test_game.py` (修改), `test_integration.py` (修改)
- **变更说明**:
  - **`run()` 解耦重构**:
    - 渲染帧率控制：`renderer.tick()` 以 RENDER_FPS=60 驱动，返回 ms 级 dt
    - 时间累加器：`accumulator += dt`，当 >= `_get_current_tick_interval()` 时触发 `_update()`
    - spiral-of-death 保护：MAX_CATCHUP_STEPS=5，超限时丢弃多余 accumulator 并记录 DEBUG 日志
    - 非 RUNNING 或暂停时 accumulator 清零，不推进逻辑
    - 每帧调用 `_update_boost_state(dt, command['boost'])` 进行平滑加速过渡
    - `draw_frame` 传递 `is_boosting=self.snake.is_boosting` 参数
  - **新增 `_update_boost_state(dt, boost_pressed)`**:
    - 目标倍率：BOOST_SPEED_MULTIPLIER(按) 或 1.0(松)，150ms 线性插值平滑过渡
    - BOOST_TRANSITION_SECONDS <= 0 时瞬时切换；防御性 clamp [1.0, MAX_BOOST_MULTIPLIER]
    - is_active = multiplier > 1.01
  - **新增 `_get_current_tick_interval()`**:
    - interval = BASE_TICK_INTERVAL / multiplier，下限 20ms
    - NaN/零/负值防御：回退到 BASE_TICK_INTERVAL 并记录 ERROR
  - **新增 `_reset_boost()`**: 强制复位 boost_state 到初始值
  - **`_update()` 修改**: 三个 GAME_OVER/VICTORY 分支 return 前各追加 `self._reset_boost()`
  - **`reset()` 修改**: 末尾追加 `self._reset_boost()` 作为双重保险
  - **test_integration.py 修复**: 3 个 FPS 测试目标从 FPS=10 更新为 RENDER_FPS=60；4 个 VICTORY 测试食物位置从 (19,15) 修正为 (20,15)（蛇头前方一格）
