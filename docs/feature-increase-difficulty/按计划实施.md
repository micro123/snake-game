# 实施记录

> **功能**: 分数递增难度（Score-Based Difficulty Scaling） | **阶段**: 实施 (P4) | **角色**: 开发工程师 | **日期**: 2026-07-10

## 1. 实施摘要

在现有 MVC 协调器架构上实现分数驱动难度递增功能。核心机制：每吃 N 个食物（SCORE_THRESHOLD_INTERVAL=50 分）提升一个难度等级，每个等级降低 tick 间隔（加速 DIFFICULTY_INCREMENT=0.1 倍），与按键加速（boost）乘积叠加生效，MIN_TICK_INTERVAL=20ms 硬下限。HUD 居中显示 Lv.N / Speed Mx。共变更 5 个源文件 + 3 个测试文件，新增约 99 项测试，全量 680 测试零回归通过。

### T-003 Game — 修改 _get_current_tick_interval() 公式

T-003 implementation complete. Here is the summary:

**summary**: 修改 `_get_current_tick_interval()` 公式，从单因子 `BASE / boost` 扩展为复合叠加 `BASE / (1 + difficulty_multiplier + boost_extra)`，其中 `boost_extra = boost_multiplier - 1.0`。向后兼容：`difficulty_multiplier=0` 时公式等价于旧版。施加 `max(result, MIN_TICK_INTERVAL)` 硬下限。保留 try/except 异常捕获并回退到 100ms。新增 11 项复合公式测试，全量 610 测试通过。变更文件 2 个（game.py、test_game.py）。
