```python
def is_boost_pressed(self) -> bool:
    if self._paused:
        # 暂停状态下不激活加速（即使物理按键被按住）
        return False
    keys = pygame.key.get_pressed()
    if keys[BOOST_KEY]:
        self._boost_active = True
        return True
    # 注：释放检测不在此处——当 get_pressed()[BOOST_KEY]==False 时 _boost_active 是否清零
    # 取决于是否处于暂停后恢复的边界场景。正常释放时，下一帧轮询即检测到 False。
    return False
```

### 1.4 game.py

现有类 `Game`，核心重构 + 新增方法：

| 新增/变更方法 | 签名 | 说明 |
|---|---|---|
| **重构** `run()` | `run() -> None` | 解耦渲染与逻辑：渲染以 `RENDER_FPS` 驱动；逻辑通过时间累加器按可变 `tick_interval` 推进 |
| **新增** | `_update_boost_state(dt: float, boost_pressed: bool) -> None` | 每帧调用，根据 `boost_pressed` 计算目标倍率并更新 `snake.boost_state`（含平滑过渡逻辑） |
| **新增** | `_get_current_tick_interval() -> int` | 返回当前逻辑 tick 间隔：`BASE_TICK_INTERVAL / snake.boost_multiplier`，结果 clamp 到 `>= 20` ms |
| **新增** | `_reset_boost() -> None` | 复位 boost 状态：`snake.boost_state` 重置为初始值，`input_handler._boost_active` 重置为 `False` |
| **变更** `_update()` | `_update() -> None` | 保持逻辑不变，但调用时机由时间累加器触发而非每帧固定执行 |
| **变更** `_handle_command()` | `_handle_command(command: dict) -> bool` | 处理 `command['boost']`（在 `_update_boost_state` 中消费）；`restart` 命令触发 `_reset_boost()` |
| **变更** `reset()` | `reset() -> None` | 在现有逻辑末尾调用 `_reset_boost()` |

**解耦的 `run()` 主循环（伪代码）**：

```python
def run(self) -> None:
    accumulator = 0.0  # ms
    running = True
    while running:
        dt = self.renderer.tick()  # 返回 ms，内部以 RENDER_FPS 限速

        command = self.input_handler.process_events(
            self.snake.direction, self.state
        )
        running = self._handle_command(command)

        # 更新 boost 状态（每帧都调用，含平滑过渡）
        self._update_boost_state(dt, command.get('boost', False))

        # 仅 RUNNING 且未暂停时推进逻辑
        if self.state == GameState.RUNNING and not self.input_handler.is_paused():
            tick_interval = self._get_current_tick_interval()
            accumulator += dt
            steps = 0
            while accumulator >= tick_interval and steps < MAX_CATCHUP_STEPS:
                self._update()  # 原有逻辑方法
                accumulator -= tick_interval
                # 若 _update 改变了状态（GAME_OVER/VICTORY），立即中断
                if self.state != GameState.RUNNING:
                    accumulator = 0.0
                    break
                steps += 1
                # 重新计算 tick_interval（平滑过渡期间可能每步不同）
                tick_interval = self._get_current_tick_interval()
        else:
            accumulator = 0.0  # 非 RUNNING/暂停时清空累加器

        self.renderer.draw_frame(
            self.snake, self.food, self.score, self.state
        )
        pygame.display.flip()
```

**`_update_boost_state(dt, boost_pressed)` 平滑过渡逻辑**：

```python
def _update_boost_state(self, dt: float, boost_pressed: bool) -> None:
    # 非 RUNNING 状态下忽略加速输入
    if self.state != GameState.RUNNING:
        self.snake.boost_state['is_active'] = False
        self.snake.boost_state['current_multiplier'] = 1.0
        return

    target = BOOST_SPEED_MULTIPLIER if boost_pressed else 1.0
    current = self.snake.boost_state['current_multiplier']

    # 线性插值平滑过渡
    if abs(target - current) < 0.01:
        self.snake.boost_state['current_multiplier'] = target
        self.snake.boost_state['is_active'] = (target > 1.0)
        return

    step = dt / (BOOST_TRANSITION_SECONDS * 1000.0)
    diff = target - current
    if abs(diff) <= step:
        new_multiplier = target
    else:
        new_multiplier = current + (step if diff > 0 else -step)

    # clamp 到安全范围
    new_multiplier = max(1.0, min(MAX_BOOST_MULTIPLIER, new_multiplier))
    self.snake.boost_state['current_multiplier'] = new_multiplier
    self.snake.boost_state['is_active'] = (new_multiplier > 1.01)

    # 若 boost_pressed 为 False 但当前 multiplier 已接近 1.0，确保完全复位
    if not boost_pressed and new_multiplier <= 1.05:
        self.snake.boost_state['current_multiplier'] = 1.0
        self.snake.boost_state['is_active'] = False
```

**`_get_current_tick_interval()` 逻辑**：

```python
def _get_current_tick_interval(self) -> int:
    multiplier = self.snake.boost_multiplier
    # 防御：multiplier 必须 >= 1.0 以防止除零/负值
    if multiplier < 1.0:
        multiplier = 1.0
    raw = BASE_TICK_INTERVAL / multiplier
    # clamp 最短 tick 间隔为 20ms（对应 50Hz，即 5x 倍率时 100/5=20ms）
    return max(20, int(raw))
```

**碰撞 / 胜利后的即时 boost 复位**：

在 `_update()` 方法中，`GAME_OVER` 和 `VICTORY` 状态设置后立即调用 `_reset_boost()`：

```python
# _update() 中边界碰撞分支末尾:
if self.snake.check_boundary_collision(GRID_COLS, GRID_ROWS):
    self.state = GameState.GAME_OVER
    self._reset_boost()
    return

# _update() 中自撞分支末尾:
if self.snake.check_self_collision():
    self.state = GameState.GAME_OVER
    self._reset_boost()
    return
```

    head_color = BOOST_SNAKE_HEAD if is_boosting else COLORS.SNAKE_HEAD

## 4. 关键流程

### 4.1 基础加速流程：按下加速 -> 蛇加速 -> 释放恢复

```
[键盘] SPACE 按下
    |
    v
[InputHandler.is_boost_pressed()] -- poll pygame.key.get_pressed()[BOOST_KEY] --> True
    |
    v
[Game._update_boost_state(dt, boost_pressed=True)]
    |  target = BOOST_SPEED_MULTIPLIER (2.0)
    |  current (1.0) -> 线性插值 -> 逐步逼近 2.0
    |  每帧更新 snake.boost_state['current_multiplier']
    |
    v
[Game._get_current_tick_interval()]
    |  return BASE_TICK_INTERVAL / snake.boost_multiplier
    |  e.g. 100ms / 2.0 = 50ms
    |
    v
[Game.run() 时间累加器]
    |  accumulator += dt  (dt = ~16.67ms @ 60fps)
    |  while accumulator >= tick_interval (50ms):
    |      _update()  -- 蛇前进一步
    |      accumulator -= tick_interval
    |
    v  ... 几帧后 ...
[键盘] SPACE 释放
    |
    v
[InputHandler.is_boost_pressed()] --> False
    |
    v
[Game._update_boost_state(dt, boost_pressed=False)]
    |  target = 1.0
    |  current 从 2.0 -> 线性插值 -> 逐步回退到 1.0
    |
    v
[Game._get_current_tick_interval()]
    |  return 100ms / 1.0 = 100ms (恢复基准速度)
```

### 4.2 加速中 Game Over 流程

```
[Game._update()] -- 单步逻辑执行
    |  check_boundary_collision() or check_self_collision() --> True
    |
    v
[Game]
    |  self.state = GameState.GAME_OVER
    |  self._reset_boost()  -- 立即复位:
    |      snake.boost_state = {'is_active': False, 'current_multiplier': 1.0}
    |      input_handler._boost_active = False
    |
    v
[Game.run()] accumulator = 0.0  (break 中断)
    |
    v
[Renderer.draw_frame(..., is_boosting=False)]
    |  draw_snake: 使用 COLORS.SNAKE_BODY / HEAD (正常颜色)
    |  draw_hud: 不渲染 "BOOST" 文字
    |  draw_game_over: 叠加结束遮罩
```

### 4.3 加速中暂停 / 窗口失焦流程

```
[系统] Alt+Tab / 点击其他窗口
    |
    v
[Pygame] WINDOWFOCUSLOST 事件入队
    |
    v
[InputHandler.process_events()]
    |  event.type == pygame.WINDOWFOCUSLOST:
    |      self._paused = True
    |      self._boost_active = False   -- 强制清除加速标志
    |      last_action = 'pause'
    |
    v
[Game.run()]
    |  command['action'] = 'pause'
    |  command['boost'] = False (因 _boost_active 已清除)
    |  _update_boost_state(dt, False) --> target = 1.0, 开始减速过渡
    |  is_paused() = True --> accumulator = 0.0, 不执行 _update()
    |
    v
[恢复时]
    |  即使 SPACE 仍被物理按住，is_boost_pressed() 返回 False
    |  (因为 get_pressed() 中键状态需要一次释放-重新按下才会刷新)
    |  玩家需释放后重新按 SPACE 才能再次激活加速
```

### 4.4 加速中重启流程

```
[玩家] R 键 (在 GAME_OVER / VICTORY 状态下)
    |
    v
[InputHandler.process_events()]
    |  action = 'restart'
    |
    v
[Game._handle_command()]
    |  action == 'restart' -> self.reset()
    |
    v
[Game.reset()]
    |  snake.reset()     -- 已有逻辑 + 内部 reset 复位 boost_state
    |  food.reset()      -- 已有逻辑
    |  food.respawn()    -- 已有逻辑
    |  score = 0         -- 已有逻辑
    |  state = RUNNING   -- 已有逻辑
    |  _reset_boost()    -- 新增：双重保险确保 boost 复位
    |
    v
[结果] 以基准速度开始新游戏，boost_state 完全复位
```

### 4.5 平滑过渡流程（FR-007，P2 优先级）

```
按下 SPACE 的瞬间 (target = 2.0, current = 1.0):
    Frame 0:  current_multiplier = 1.0   tick_interval = 100ms
    Frame 1:  dt=16.67ms, step=16.67/150=0.111, new = 1.111  tick_interval = 90ms
    Frame 2:  new = 1.222  tick_interval = 82ms
    ...约 9 帧 (150ms) 后达到 2.0...
    Frame 9+: current = 2.0  tick_interval = 50ms  (稳定加速)

释放 SPACE 的瞬间 (target = 1.0, current = 2.0):
    对称地，约 9 帧后回到 1.0，tick_interval 恢复 100ms
```

## 5. 错误处理

### 5.1 错误码定义

| 错误码 | HTTP 状态 | 说明 | 可重试 | 处理策略 |
|---|---|---|---|---|
| `CFG-001` | N/A | `BOOST_SPEED_MULTIPLIER` 超出 [1.0, MAX_BOOST_MULTIPLIER] | 是（修正配置后重启） | clamp 到边界值，`logging.warning()` |
| `CFG-002` | N/A | `BOOST_KEY` 无效键码 | 是（修正配置后重启） | 回退为 `pygame.K_SPACE`，`logging.warning()` |
| `CFG-003` | N/A | `BASE_TICK_INTERVAL` <= 0 | 是（修正配置后重启） | 重置为 100，`logging.warning()` |
| `RT-001` | N/A | tick_interval 计算异常（NaN/负值/零值） | 否（自动回退） | 回退到 `BASE_TICK_INTERVAL`，`logging.error()` |
| `RT-002` | N/A | 单帧 catch-up 步数达到上限 | 否（自动处理） | 丢弃多余累加器时间，`accumulator = 0`，`logging.debug()` |
| `RT-003` | N/A | 平滑过渡计算除零（BOOST_TRANSITION_SECONDS=0） | 否（自动回退） | 瞬时切换（无过渡），`logging.debug()` |

### 5.2 日志策略

| 级别 | 触发条件 | 日志内容示例 |
|---|---|---|
| `WARNING` | 配置参数校验失败，已自动修正 | `"BOOST_SPEED_MULTIPLIER=25.0 clamped to MAX_BOOST_MULTIPLIER=5.0"` |
| `WARNING` | 无效 BOOST_KEY 回退 | `"BOOST_KEY=999 out of range, fallback to pygame.K_SPACE"` |
| `ERROR` | tick_interval 计算出非法值 | `"tick_interval computed as {value}, fallback to BASE_TICK_INTERVAL"` |
| `DEBUG` | catch-up 步数达到上限 | `"Max catchup steps ({MAX_CATCHUP_STEPS}) reached, discarding {excess}ms"` |
| `DEBUG` | 平滑过渡因 TRANSITION_SECONDS=0 而跳过 | `"BOOST_TRANSITION_SECONDS=0, using instant transition"` |

### 5.3 降级策略

| 异常场景 | 降级行为 | 影响范围 |
|---|---|---|
| `BOOST_SPEED_MULTIPLIER` 异常值 | clamp 到安全范围 [1.0, MAX_BOOST_MULTIPLIER] | 加速倍率受限，游戏仍可玩 |
| `BOOST_KEY` 无效 | 回退为 `pygame.K_SPACE` | 键位不可自定义，功能仍可用 |
| tick_interval 计算出非法值 | 回退为 `BASE_TICK_INTERVAL` | 当前帧以基准速度运行，下一帧恢复 |
| 焦点丢失导致按键残留 | `_boost_active = False` 强制清除 | 切回后需释放再按下，体验略受影响但安全 |
| 平滑过渡除零 | 瞬时切换（当前 multiplier 直接 = target） | 失去平滑过渡视觉效果，功能仍正常 |
| 极端加速导致单帧需要执行过多逻辑步 | catch-up 步数上限保护，丢弃多余时间 | 可能短暂丢帧，但避免假死 |
| Pygame 输入法拦截加速键 | `get_pressed()` 检测不到按键 | 该帧不加速，释放后可重试 |
