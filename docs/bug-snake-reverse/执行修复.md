# 修复记录

> **Bug**: snake-reverse | **阶段**: 执行修复 (P3) | **角色**: 开发工程师 | **日期**: 2026-07-10

## 1. 修复摘要

在 Snake 类中新增 `committed_direction` 属性，仅在 `move_and_grow()`（蛇实际物理移动）时更新。将 `process_events()` 和 `change_direction()` 的反向检测基准从可变的 `self.direction` 改为 `committed_direction`，消除跨帧方向漂移导致的逆向绕过问题。

**snake.py (`__init__`)**：
```python

```

**snake.py (`_init_body`)**：
```python

```

**snake.py (`move_and_grow`)**：
```python
self.committed_direction = self.direction  # 记录本次物理移动方向
```

**snake.py (`change_direction`)**：
```python
# 反向拦截：基于 committed_direction（上次物理移动方向）检测
if (dx + self.committed_direction[0] == 0) and (dy + self.committed_direction[1] == 0):
    return False
```

**game.py (`run` 第 110-111 行)**：
```python
command = self.input_handler.process_events(
    self.snake.committed_direction, self.state

```

## 6. 验收结果

- **Bug 复现场景验证**: 新增的 4 个集成测试 (`TestCrossFrameReverseIntercept`) 精确模拟 Bug 触发条件 — 蛇向上移动时帧1按LEFT、帧2按DOWN（无物理更新间），DOWN 被正确拦截 (action='none')。原有的合法 L 形转弯同样测试通过。
- **全量回归测试**: 670 passed, 0 failed, 1 warning（AVX2 无关警告）
- **回归风险检查**: 对所有 `snake.direction` 和 `self.direction` 引用进行了审计 — `move_and_grow` 中使用 direction 做移动计算（正确）、`change_direction` 中使用 direction 存储新方向（正确）、`game.py` 中使用 direction 做 next_head 预计算（在 _update 中 move_and_grow 之前调用，方向一致）。无遗漏应改用 committed_direction 的引用。
- **验收结论**: Bug 已修复，无回归问题。`committed_direction` 机制在物理帧之间保持稳定，防止了跨帧方向漂移导致的逆向绕过。`change_direction` 的 docstring 已同步更新以反映新逻辑。
