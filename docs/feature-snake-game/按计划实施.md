# 实施记录

> **功能**: 经典贪吃蛇游戏 (Snake Game) | **阶段**: 实施 (P5) | **角色**: 开发工程师 | **日期**: 2026-07-08

## 1. 实施摘要

按照详细设计文档的 9 个任务分 6 个阶段实施，完成经典贪吃蛇游戏的全部功能开发。最终交付 14 个源文件 + 7 个测试文件，共 373 个测试用例全部通过。覆盖全部 9 条验收标准、9 个边界情况、4 项风险缓解验证。

## 2. 任务执行记录

### T-001 创建 config.py 常量模块

- **变更文件**: `config.py` (新建)
- **变更说明**: 定义窗口尺寸(800x600)、网格参数(40x30, CELL_SIZE=20)、游戏参数(FPS=10/初始蛇长3/每食物10分)、HUD高度40px 及 COLORS 类(7色)。模块无 Pygame 依赖，全部常量值与设计规格一致，自动化断言验证通过。
- **测试**: 项目无测试框架，跳过测试。

### T-002 实现 Snake 类核心逻辑

- **变更文件**: `snake.py` (新建), `test_snake.py` (新建)
- **变更说明**: 创建 `snake.py` 实现 Snake 类核心逻辑（含 `__init__`、`head`/`length` property、`move_and_grow` 合二为一操作、`change_direction` 反向拦截、`check_boundary_collision` 边界检测、`check_self_collision` 自撞检测、`reset` 重置），共计 120 行纯 Python 逻辑代码。编写 `test_snake.py` 单元测试文件，使用内置 unittest 框架，覆盖 6 个测试类共 46 个用例，全部通过。
- **测试**: 46 个单元测试用例全部通过。

### T-003 实现 Food 类

- **变更文件**: `food.py` (新建), `test_food.py` (新建)
- **变更说明**: 创建 `food.py`（Food 类，22 行）与 `test_food.py`（24 个单元测试用例）。Food 类实现了集合差运算的食物随机生成 (`respawn`)、满棋盘返回 False（胜利条件）、占位初始值与重置。24 个测试全部通过，Snake 的 46 个回归测试也全部通过。
- **测试**: 24 个单元测试用例全部通过。

### T-004 实现 Renderer 渲染层

- **变更文件**: `renderer.py` (新建), `game.py` (新建), `test_renderer.py` (新建)
- **变更说明**:
  - `renderer.py`: Renderer 渲染类，封装 8 个绘制方法与 tick 帧率控制。使用 `pygame._freetype` (C 扩展) 进行字体渲染，规避 pygame 2.6.1 + Python 3.14 中 `pygame.font`/`pygame.freetype`/`pygame.sysfont` 三模块间的循环导入 bug。
  - `game.py`: GameState 枚举 (RUNNING/GAME_OVER/VICTORY)，供渲染层依赖。
  - `test_renderer.py`: 44 个单元测试用例，覆盖 10 个测试类。
  - 图层渲染顺序: background -> snake(tail to head) -> food(placeholder skip) -> HUD -> conditional overlay。
  - 蛇头(SNAKE_HEAD=50,255,50)与蛇身(SNAKE_BODY=0,200,0)颜色明确区分。
  - 半透明遮罩通过 `pygame.SRCALPHA` Surface 实现 (OVERLAY_BG=0,0,0,180)。
- **测试**: 44 个单元测试用例全部通过。

### T-005 实现 InputHandler 事件处理

- **变更文件**: `input_handler.py` (新建), `test_input_handler.py` (新建)
- **变更说明**: 新建 `input_handler.py`（106行）和 `test_input_handler.py`（67测试用例），覆盖方向映射/反向拦截/Quit事件/RQ键状态门控/per-frame方向缓冲/焦点暂停/空队列/非方向键忽略/边界行为。方向键映射为正确方向向量并拦截反向，QUIT事件返回quit动作，R键仅GAME_OVER/VICTORY时返回restart，Q键仅GAME_OVER/VICTORY时返回quit，同帧多方向键取最后有效者，焦点丢失暂停/恢复继续。Pygame实际API使用独立事件类型 `WINDOWFOCUSLOST`/`WINDOWFOCUSGAINED` 而非 `WINDOWEVENT` 子类型，已据此调整实现和测试。
- **测试**: 67 个测试用例全部通过。

### T-006 实现 Game 协调器与状态机

- **变更文件**: `game.py` (修改), `main.py` (修改), `test_game.py` (新建)
- **变更说明**: `game.py` 包含完整的 `Game` 类（含 `__init__` 创建窗口+实例化全部组件+生成首个食物、`run` 主循环 tick->process_events->_handle_command->_update->draw_frame->flip、`_handle_command` 分发 quit/restart/direction、`_update` 食物碰撞/增长加分/respawn/边界自撞检测/胜利状态迁移、`reset` 全部组件重置）及 `GameState` 枚举重新导出。`main.py` 提供 Pygame 依赖检测和入口。`test_game.py` 含 55 个单元测试，全部 236 个测试通过。
- **测试**: 55 个单元测试用例全部通过，累计 236 个测试通过。

### T-007 实现 main.py 入口点
